Este documento descreve o fluxo de execução do projeto,
explicando o papel de cada arquivo Python e como eles se
conectam. O arquivo central de controle é o main.py.

============================================================
1. main.py (O Orquestrador)
============================================================
Função: Ponto de entrada da pipeline. Configura o ambiente
(cria as pastas 'models/' e 'results/') e encadeia a
execução das fases de Dados, Treinamento, Avaliação e Análise.

============================================================
2. FASE DE DADOS E PRÉ-PROCESSAMENTO
============================================================

2.1. loader.py
Função: Carrega os dados brutos (CSV ou dataset nativo do scikit-learn).

2.2. preprocess.py
Função: Aplica transformações essenciais: remoção de colunas,
codificação do target (B/M para 0/1), divisão treino/teste e
padronização das features (StandardScaler).

2.3. describe_dataset.py
Função: Gera relatórios de Análise Exploratória (EDA).
Ações: Calcula e salva estatísticas descritivas, matriz de
correlação e gráficos de distribuição/histogramas (em results/).

============================================================
3. FASE DE MODELOS E TREINAMENTO
============================================================

3.1. model_hyperparameters.py
Função: Define os modelos base (ex: LogisticRegression, KNN)
e suas configurações iniciais, incluindo a grade de
hiperparâmetros para otimização futura.

3.2. train.py
Função: Orquestra o treinamento.
Ações: Carrega/pré-processa dados, treina os modelos definidos,
e salva todos os artefatos de ML (modelos, scaler e test_data)
na pasta 'models/'.

3.3. model_utils.py
Função: Utilitário para carregar modelos treinados ('*.joblib')
do disco para as fases de avaliação e análise.

============================================================
4. FASE DE AVALIAÇÃO E ANÁLISE (RELATÓRIOS)
============================================================

4.1. evaluate.py
Função: Avalia o desempenho dos modelos no conjunto de teste.
Ações: Calcula métricas (F1, AUC, Accuracy), gera a Tabela
Comparativa (CSV) e plota o gráfico de Curvas ROC comparativas.

4.2. predict_and_visualize.py
Função: Executa as previsões finais.
Ações: Gera a tabela consolidada de previsões de todos os
modelos ('predictions_table.csv'), necessária para a análise de erros.

4.3. error_analysis_table.py
Função: Detalha os tipos de erro.
Ações: Calcula e salva o resumo de erros (FP vs FN) e os
casos de erro detalhados, além de gerar o gráfico resumo.

4.4. feature_importance.py
Função: Analisa e visualiza a relevância das features
para os modelos que suportam este cálculo.

============================================================
FLUXO RESUMIDO DO PROJETO (Ordem de Dependência)
============================================================

loader.py/preprocess.py -> train.py (cria modelos/)
    -> describe_dataset.py (analisa test_data)

train.py (modelos/ pronto)
    -> evaluate.py (gera métricas e ROC)

evaluate.py
    -> predict_and_visualize.py (gera predictions_table.csv)
        -> error_analysis_table.py (analisa FP vs FN)

============================================================
Observações importantes
============================================================

- A pasta 'models/' armazena artefatos de ML.
- A pasta 'results/' armazena relatórios (CSV/PNG).
- Ambas as pastas devem ser ignoradas pelo Git.
